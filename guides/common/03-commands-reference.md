# 커맨드 퀵 레퍼런스

20개 커맨드를 한눈에 보는 참고서입니다.

---

## 필수 커맨드 (프로젝트 완성 플로우)

| # | 커맨드 | 하는 일 | 소요시간 | 입력 | 출력 |
|---|--------|---------|----------|------|------|
| 1 | `/hello` | 프로젝트 인프라 셋업 | 2-3분 | 프로젝트명, 설명, 스택 | 폴더 + GitHub 저장소 |
| 2 | `/new` | 아이디어 입력 + 팀 자동 구성 | 1-2분 | 프로젝트 아이디어 | 팀 구성 + 모드 선택 |
| 3 | `/discuss` | 팀 토론 → 기획서 작성 | 3-10분 | (자동) | 기획서 |
| 4 | `/approve` | 기획서 승인 + 작업 분배 | 1-2분 | 승인/수정/재토론 | 작업 목록 |
| 5 | `/execute` | 작업 실행 + 크로스 리뷰 | 5-15분 | 실행 모드 선택 | 완성된 결과물 |
| 6 | `/report` | 최종 보고서 생성 | 1분 | (자동) | 마크다운 보고서 |

**기본 플로우:** `/hello` → `/new` → `/discuss` → `/approve` → `/execute` → `/report`

> `/hello` 없이 `/new`로 바로 시작할 수도 있습니다. (인프라 셋업 불필요 시)

## 보조 커맨드 (프로젝트 관리)

| 커맨드 | 하는 일 | 언제 사용? |
|--------|---------|-----------|
| `/status` | 현재 프로젝트 대시보드 | 진행 상황 확인 시 |
| `/feedback` | 팀원 성과 분석 → 에이전트 개선 | 프로젝트 완료 후 |
| `/my-team` | 팀원 정보 + 역할 카탈로그 | 팀 구성 확인 시 |
| `/learn` | 역할별 학습 가이드 | 더 배우고 싶을 때 |

## 고급 커맨드 (커스터마이징)

| 커맨드 | 하는 일 | `/new`와 차이 |
|--------|---------|-------------|
| `/new-project` | 수동 프로젝트 생성 | 타입/모드를 직접 선택 |
| `/projects` | 전체 프로젝트 목록 | `/status`는 현재 프로젝트만 |
| `/onboarding` | 사용자 환경 설정 (1회성) | `/hello`는 프로젝트별 |

| 커맨드 | 하는 일 |
|--------|---------|
| `/my-config` | 현재 설정 대시보드 |
| `/scaffold` | 프로젝트 템플릿 스캐폴딩 (5개 내장) |
| `/add-skill` | 스킬 추가 |
| `/add-agent` | 에이전트 추가 |
| `/preset` | 프리셋 관리 (역할/스택/워크플로우) |
| `/reset` | 설정 초기화 |
| `/eval` | A/B 평가 (멀티에이전트 vs 단순 접근법) |

## 모드별 플로우 비교

| 모드 | 팀 규모 | 토론 | 승인 | 실행 | 추천 상황 |
|------|---------|------|------|------|----------|
| quick-build | 2-3명 | 없음 | 자동 | O | 간단한 봇, 스크립트 |
| plan-execute | 3-5명 | 1라운드 | 자동 | O | 웹앱, API 서버 |
| plan-only | 5-15명 | 최대 3라운드 | CEO 승인 | O | 대규모 시스템 |

## 혼동하기 쉬운 커맨드

| 비교 | 차이 |
|------|------|
| `/new` vs `/new-project` | 자동(복잡도 분석 → 모드 추천) vs 수동(직접 선택) |
| `/hello` vs `/onboarding` | 프로젝트 인프라 셋업 vs 사용자 환경 설정(1회성) |
| `/status` vs `/projects` | 현재 프로젝트 대시보드 vs 전체 프로젝트 목록 |
