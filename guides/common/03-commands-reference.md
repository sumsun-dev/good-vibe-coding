# 커맨드 레퍼런스

Good Vibe Coding에는 20개의 슬래시 커맨드가 있습니다.
처음에는 6개만 알면 충분하고, 나머지는 필요할 때 찾아 쓰면 됩니다.

---

## 기본 플로우 (6개)

프로젝트 하나를 처음부터 끝까지 만드는 순서입니다.

| # | 커맨드 | 뭘 하나요? | 걸리는 시간 | 뭘 넣고 뭘 받나요? |
|---|--------|-----------|-----------|-------------------|
| 1 | `/hello` | 프로젝트 세팅 | 2-3분 | 프로젝트명, 설명, 스택 → 폴더 + GitHub 저장소 |
| 2 | `/new` | 팀 구성 | 1-2분 | 아이디어 → 팀 구성 + 모드 선택 |
| 3 | `/discuss` | 팀 토론 | 3-10분 | (자동) → 기획서 |
| 4 | `/approve` | 기획서 승인 | 1-2분 | 승인/수정/재토론 → 작업 분배 |
| 5 | `/execute` | 실행 + 리뷰 | 5-15분 | 실행 모드 선택 → 완성된 결과물 |
| 6 | `/report` | 보고서 | 1분 | (자동) → 마크다운 보고서 |

```
/hello  →  /new  →  /discuss  →  /approve  →  /execute  →  /report
```

> `/hello` 없이 `/new`로 바로 시작할 수도 있습니다. 폴더/저장소가 이미 있을 때.

## 관리 커맨드 (4개)

프로젝트 진행 중에 필요하면 쓰는 커맨드입니다.

| 커맨드 | 뭘 하나요? | 언제 쓰나요? |
|--------|-----------|-------------|
| `/status` | 프로젝트 현황판 | 진행 상황 확인할 때 |
| `/feedback` | 팀원 성과 분석 + 개선 제안 | 프로젝트 끝난 후 |
| `/my-team` | 팀원 정보 + 역할 카탈로그 | 팀 구성 확인할 때 |
| `/learn` | 역할별 학습 가이드 | 사용법 더 알고 싶을 때 |

## 모드 비교

`/new`에서 복잡도를 분석한 결과에 따라 자동 추천됩니다.

| 모드 | 팀 규모 | 토론 | 이런 프로젝트에 적합 |
|------|---------|------|---------------------|
| **quick-build** | 2-3명 | 생략 | 간단한 봇, 스크립트 |
| **plan-execute** | 3-5명 | 1라운드 | 웹앱, API 서버 |
| **plan-only** | 5-8명 | 최대 3라운드 | 대규모 시스템 |

## 실행 중 문제가 생기면?

`/execute` 도중 품질 리뷰에서 문제가 발견되면, 자동으로 처리됩니다.

| 순서 | 동작 |
|------|------|
| 1 | 문제를 카테고리별로 분류 (보안, 빌드, 테스트, 성능, 타입, 아키텍처, 로직) |
| 2 | 이전 시도에서 뭘 했고 뭐가 안 됐는지를 포함해서 수정 요청 (최대 2번) |
| 3 | 그래도 안 되면 당신(CEO)에게 물어봄 — 계속 / 건너뛰기 / 중단 |

수동으로 개입할 필요 없이 대부분 자동으로 해결됩니다.

## 헷갈리기 쉬운 것들

| 이거랑 | 저거는 | 이렇게 다릅니다 |
|--------|--------|----------------|
| `/new` | `/new-project` | `/new`는 복잡도를 보고 알아서 추천, `/new-project`는 직접 고름 |
| `/hello` | `/onboarding` | `/hello`는 프로젝트마다 하는 세팅, `/onboarding`은 최초 1회 환경 설정 |
| `/status` | `/projects` | `/status`는 현재 프로젝트만, `/projects`는 전체 목록 |

## 에이전트용 내부 API

에이전트가 내부적으로 사용하는 CLI 커맨드가 101개 있습니다.
일반 사용자는 쓸 일이 없지만, 커스텀 에이전트를 만들 때 참고할 수 있습니다.

```bash
# 특정 커맨드의 입출력 스키마 확인
node ${CLAUDE_PLUGIN_ROOT}/scripts/cli.js describe-command --command init-execution

# 전체 목록
node ${CLAUDE_PLUGIN_ROOT}/scripts/cli.js describe-command
```
