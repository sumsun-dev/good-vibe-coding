# Claude Code 기본 사용법

## 대화 시작하기

터미널에서 `claude`를 입력하면 대화가 시작됩니다.

```bash
# 기본 시작
claude

# 특정 디렉토리에서 시작
cd ~/my-project && claude
```

## 기본 요청 패턴

### 파일 관련

```
이 폴더의 구조를 보여줘
README.md 파일을 읽어줘
config.json 파일에서 port 설정을 3000에서 8080으로 변경해줘
src/ 폴더에서 "TODO" 라고 쓴 곳을 모두 찾아줘
```

### 정보 검색

```
이 프로젝트의 기술 스택이 뭐야?
package.json에서 사용하는 의존성을 설명해줘
이 에러 메시지가 무슨 뜻이야? [에러 메시지 붙여넣기]
```

### 작업 요청

```
새 컴포넌트를 만들어줘: 로그인 폼
이 함수에 테스트를 작성해줘
이 코드를 TypeScript로 변환해줘
이 CSV 파일을 분석해서 요약해줘
```

## 커맨드 활용

슬래시(/)로 시작하는 커맨드로 빠르게 작업할 수 있습니다:

```
good-vibe:onboarding    → 초기 설정 마법사
good-vibe:learn         → 학습 가이드
good-vibe:my-config     → 현재 설정 확인
/help                   → 도움말
```

## 효과적인 요청 팁

### 1. 구체적으로 요청하기

```
"이거 고쳐줘"
"login.js 파일의 handleSubmit 함수에서 비밀번호 검증이 동작하지 않아. 8자 이상인지 확인하는 로직을 추가해줘"
```

### 2. 맥락 제공하기

```
"테스트 만들어줘"
"utils/format.js의 formatDate 함수에 대한 단위 테스트를 vitest로 만들어줘. 한국 시간대(KST) 케이스도 포함해줘"
```

### 3. 단계적으로 진행하기

```
1단계: "이 프로젝트 구조를 분석해줘"
2단계: "사용자 인증 기능을 추가하려면 어떤 파일들을 수정해야 해?"
3단계: "그러면 먼저 로그인 API 엔드포인트를 만들어줘"
```

### 4. 역할별 활용 예시

PM/기획자:

```
이 기획서의 빠진 부분을 찾아줘
이슈 목록을 우선순위별로 정리해줘
이 회의록에서 액션 아이템을 추출해줘
```

디자이너:

```
이 CSS에서 접근성 문제가 있는지 확인해줘
디자인 토큰을 CSS 변수로 변환해줘
이 컴포넌트의 반응형 스타일을 추가해줘
```

콘텐츠 크리에이터:

```
이 블로그 글의 SEO를 개선해줘
제목 후보 5개를 만들어줘
이 글을 요약해서 SNS용 카피를 만들어줘
```

## 주의사항

- 민감한 정보: API 키, 비밀번호 등을 대화에 직접 입력하지 마세요
- 파일 수정: Claude Code는 실제 파일을 수정합니다. 중요한 파일은 Git으로 관리하세요
- 확인하기: 대규모 변경 전에 항상 현재 상태를 확인하세요
