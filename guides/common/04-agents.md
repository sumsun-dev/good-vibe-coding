# 에이전트 이해하기

## 에이전트란?

특정 역할을 수행하는 전문 AI 도우미입니다. 에이전트는 특화된 지식과 체크리스트를 가지고 있어, 해당 분야의 작업을 더 정확하고 체계적으로 수행합니다.

예를 들어, "코드를 리뷰해줘"라고 요청하면 일반적인 피드백을 받지만, 코드 리뷰어 에이전트를 사용하면 보안/성능/가독성/테스트 등 체크리스트에 따라 구조화된 피드백을 받을 수 있습니다.

---

## 에이전트 vs 스킬 차이

|           | 에이전트                     | 스킬                        |
| --------- | ---------------------------- | --------------------------- |
| 성격      | 전문가 역할 (사람처럼)       | 작업 절차 (매뉴얼처럼)      |
| 동작      | 독립적으로 판단하고 피드백   | 정해진 단계를 순서대로 실행 |
| 비유      | 전문 컨설턴트                | 업무 매뉴얼                 |
| 파일 위치 | `agents/*.md`                | `skills/*/SKILL.md`         |
| 모델      | 각 에이전트별 최적 모델 지정 | 현재 세션 모델 사용         |

## 모델 선택 기준

에이전트마다 적합한 모델이 지정되어 있습니다:

| 모델       | 특징                   | 적합한 에이전트                   |
| ---------- | ---------------------- | --------------------------------- |
| Opus   | 깊은 추론, 복잡한 판단 | 아키텍처 설계, 전략 수립          |
| Sonnet | 균형 잡힌 성능         | 코드 리뷰, 문서 검토, 데이터 분석 |
| Haiku  | 빠른 응답, 가벼운 작업 | 접근성 체크, 학습 멘토, TDD 코치  |

## 에이전트 호출 방법

### 1. 자동 호출

설정된 에이전트는 관련 작업 시 자동으로 활성화됩니다:

```
> 이 코드를 리뷰해줘          # code-reviewer-kr 활성화
> 이 문서를 검토해줘          # doc-reviewer-kr 활성화
> 접근성을 검사해줘           # accessibility-checker 활성화
```

### 2. 명시적 호출

에이전트 이름을 직접 지정할 수도 있습니다:

```
> @code-reviewer-kr 이 PR을 리뷰해줘
> @data-analyst-kr 이 CSV를 분석해줘
```

### 3. `/add-agent`로 추가

아직 설정되지 않은 에이전트를 추가할 수 있습니다:

```
> /add-agent
```

---

## 에이전트 추가하기

`/add-agent` 커맨드를 사용하면:

1. 현재 역할에 맞는 추천 에이전트 목록이 표시됩니다
2. 원하는 에이전트를 선택하면 자동으로 설치됩니다
3. `~/.claude/CLAUDE.md`에 에이전트가 등록됩니다
