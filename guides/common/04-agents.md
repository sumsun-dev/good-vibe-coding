# 에이전트 이해하기

## 에이전트란?

특정 역할을 수행하는 전문 AI 도우미입니다. 에이전트는 특화된 지식과 체크리스트를 가지고 있어, 해당 분야의 작업을 더 정확하고 체계적으로 수행합니다.

Good Vibe Coding에는 두 종류의 에이전트가 있습니다:

- **팀 에이전트 (15개)** — `good-vibe:new`에서 프로젝트를 만들면 자동으로 배정되는 AI 팀원. 토론, 구현, 리뷰를 담당합니다.
- **서포트 에이전트 (8개)** — `good-vibe:add-agent`로 직접 설치하는 개인 도우미. 코드 리뷰, TDD 코칭 등 일상 작업을 돕습니다.

예를 들어, "코드를 리뷰해줘"라고 요청하면 일반적인 피드백을 받지만, 코드 리뷰어 에이전트를 사용하면 보안/성능/가독성/테스트 등 체크리스트에 따라 구조화된 피드백을 받을 수 있습니다.

---

## 에이전트 vs 스킬 차이

|           | 에이전트                     | 스킬                        |
| --------- | ---------------------------- | --------------------------- |
| 성격      | 전문가 역할 (사람처럼)       | 작업 절차 (매뉴얼처럼)      |
| 동작      | 독립적으로 판단하고 피드백   | 정해진 단계를 순서대로 실행 |
| 비유      | 전문 컨설턴트                | 업무 매뉴얼                 |
| 파일 위치 | `agents/*.md`                | `skills/*/SKILL.md`         |
| 모델      | 각 에이전트별 최적 모델 지정 | 현재 세션 모델 사용         |

## 팀 에이전트 (15개)

`good-vibe:new`로 프로젝트를 만들면 복잡도에 따라 2-8명의 팀원이 자동 배정됩니다. 직접 설치할 필요 없이, 프로젝트 모드에 맞는 최적의 팀이 구성됩니다.

| 모드         | 팀 규모 | 주요 역할                          |
| ------------ | ------- | ---------------------------------- |
| quick-build  | 2-3명   | CTO, Backend/Fullstack, QA         |
| plan-execute | 3-5명   | CTO, PO, Fullstack, Frontend, QA   |
| plan-only    | 5-8명   | 전체 역할에서 프로젝트에 맞게 구성 |

### Leadership

| 역할          | 설명                                    |
| ------------- | --------------------------------------- |
| CTO           | 기술 아키텍처 설계, 기술 스택 결정      |
| Product Owner | 요구사항 정의, 우선순위 결정, 제품 방향 |

### Engineering

| 역할                 | 설명                          |
| -------------------- | ----------------------------- |
| Full-stack Developer | 프론트엔드 + 백엔드 전체 구현 |
| Frontend Developer   | UI 구현, 컴포넌트 설계        |
| Backend Developer    | API 설계, 비즈니스 로직 구현  |
| QA Engineer          | 테스트 전략, 품질 보증        |
| DevOps Engineer      | CI/CD, 배포, 인프라 구성      |
| Data Engineer        | 데이터 파이프라인, 분석       |
| Security Engineer    | 보안 검토, 취약점 분석        |

### Design

| 역할           | 설명                                |
| -------------- | ----------------------------------- |
| UI/UX Designer | 사용자 경험 설계, 인터페이스 디자인 |

### Support

| 역할             | 설명                               |
| ---------------- | ---------------------------------- |
| Technical Writer | 기술 문서 작성, README/가이드 정리 |

### Research

| 역할                | 설명                                    |
| ------------------- | --------------------------------------- |
| Market Researcher   | 시장 규모, 경쟁사, 트렌드 분석          |
| Business Researcher | 비즈니스 모델, 수익화, 성장 전략        |
| Tech Researcher     | 기술 스택 비교, 벤치마크, 오픈소스 분석 |
| Design Researcher   | 사용자 리서치, UX 벤치마크, 접근성 분석 |

> **팀 구성 확인**: `good-vibe:my-team`으로 현재 프로젝트의 팀 구성과 역할별 성과를 확인할 수 있습니다.

## 모델 선택 기준

에이전트마다 적합한 모델이 지정되어 있습니다:

| 모델   | 특징                   | 적합한 에이전트                               |
| ------ | ---------------------- | --------------------------------------------- |
| Opus   | 깊은 추론, 복잡한 판단 | 아키텍처 설계, 전략 수립                      |
| Sonnet | 균형 잡힌 성능         | 대부분의 팀 에이전트, 코드 리뷰, 문서 검토    |
| Haiku  | 빠른 응답, 가벼운 작업 | Tech Writer, 접근성 체크, 학습 멘토, TDD 코치 |

팀 에이전트의 모델은 역할 카테고리에 따라 자동 배정됩니다. 서포트 에이전트는 각 에이전트 파일(`agents/*.md`)에 지정된 모델을 사용합니다.

## 에이전트 호출 방법

### 팀 에이전트

프로젝트 실행 중 자동으로 호출됩니다. 직접 호출할 필요가 없습니다.

- `good-vibe:new` → 팀 자동 구성
- `good-vibe:discuss` → 역할별 분석 + 리뷰
- `good-vibe:execute` → 태스크 실행 + 크로스 리뷰

### 서포트 에이전트

설치된 서포트 에이전트는 두 가지 방법으로 호출할 수 있습니다:

**1. 자동 호출** — 관련 작업 시 자동으로 활성화됩니다:

```
> 이 코드를 리뷰해줘          # code-reviewer-kr 활성화
> 이 문서를 검토해줘          # doc-reviewer-kr 활성화
> 접근성을 검사해줘           # accessibility-checker 활성화
```

**2. 명시적 호출** — 에이전트 이름을 직접 지정합니다:

```
> @code-reviewer-kr 이 PR을 리뷰해줘
> @data-analyst-kr 이 CSV를 분석해줘
```

### 서포트 에이전트 추가

`good-vibe:add-agent`로 아직 설치하지 않은 서포트 에이전트를 추가할 수 있습니다:

```
> good-vibe:add-agent
```

---

## 서포트 에이전트 추가하기

`good-vibe:add-agent` 커맨드를 사용하면:

1. 현재 역할에 맞는 추천 서포트 에이전트 목록이 표시됩니다
2. 원하는 에이전트를 선택하면 자동으로 설치됩니다
3. `~/.claude/CLAUDE.md`에 에이전트가 등록됩니다

> **참고:** 팀 에이전트(CTO, PO 등)는 설치 불필요합니다. `good-vibe:new`로 프로젝트를 만들면 자동으로 배정됩니다.
