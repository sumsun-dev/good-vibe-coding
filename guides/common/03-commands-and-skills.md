# 커맨드와 스킬

Claude Code에서 슬래시(`/`)로 시작하는 명령어가 **커맨드**, 자동 활성화되어 작업 절차를 정의하는 가이드가 **스킬**입니다.

---

## 커맨드 (Good Vibe v2 — 6개)

Good Vibe Coding은 자연어 단일 진입점 1개와 보조 슬래시 5개로 구성됩니다. 자세한 동작은 [커맨드 레퍼런스](03-commands-reference.md) 참조.

```
/gv "..."  ← 자연어 단일 진입점 (의도 분류 + 다음 액션 안내)
   │
   ├─ /gv:status   ← 활성 프로젝트 상태 + 다음 권장 액션
   ├─ /gv:execute  ← task-graph 실행 시작
   ├─ /gv:resume   ← 중단된 실행 재개
   ├─ /gv:team     ← 팀 구성 + 모델 분포
   └─ /gv:cost     ← 예산 임계 조회/설정 (opt-in)
```

| 슬래시        | 한 줄 용도                                  | 언제 쓰나요?                     |
| ------------- | ------------------------------------------- | -------------------------------- |
| `/gv "..."`   | 자연어로 작업 던지기                        | **모든 입력의 기본 진입점**      |
| `/gv:status`  | 활성 프로젝트 + 상태별 다음 액션            | 진행 상황 빠르게 확인            |
| `/gv:execute` | 분류된 task-graph 실행                      | `/gv` 결과를 받아 실제 실행 시작 |
| `/gv:resume`  | 중단된 Phase부터 재개 (file-lock + journal) | 실행 도중 끊겼을 때              |
| `/gv:team`    | 팀 멤버 + 모델 분포                         | 누가 일하고 있는지 확인          |
| `/gv:cost`    | 예산 임계 설정/조회 (opt-in, 기본값 없음)   | 비용 한도 설정 / 누적 가시화     |

### 커맨드 입력 방법

Claude Code 대화창에서 슬래시와 함께 입력합니다:

```
> /gv 결제 시스템 만들어줘                # 자연어 진입
> /gv:status                              # 보조 슬래시
> /gv "BullMQ vs Temporal 비교"           # research 작업
> /gv "이 PR 리뷰해줘 https://..."        # review 작업
```

### v1 명령은 어떻게 됐나요?

v1의 7개 기본 명령(`hello/new/discuss/approve/execute/report/modify`)과 9개 고급 명령은 모두 v2의 `/gv` 자연어로 통합되었습니다. 상세 매핑은 [v1 → v2 마이그레이션 표](03-commands-reference.md#v1-명령-마이그레이션-표) 참조.

## 스킬

스킬은 특정 작업의 절차와 규칙을 정의한 가이드입니다.
커맨드가 "리모컨 버튼"이라면, 스킬은 "버튼을 누르면 실행되는 프로그램"입니다.

### 내장 스킬 (5개)

| 스킬                | 하는 일                      | 켜지는 시점                         |
| ------------------- | ---------------------------- | ----------------------------------- |
| `onboarding-wizard` | 대화형 온보딩 진행           | 초기 설정 / "온보딩 시작" 입력 시   |
| `project-setup`     | 프로젝트 폴더/파일 자동 생성 | `/gv` 신규 프로젝트 진입 시         |
| `multi-review`      | 멀티 AI 교차 리뷰 설정/검증  | review 작업 / `/gv:execute` 실행 시 |
| `korean-workflow`   | 역할별 워크플로우 안내       | "워크플로우 보여줘" 같은 질문 시    |
| `beginner-guide`    | 입문자 학습 가이드           | 온보딩 도중 / 첫 사용 시            |

### 스킬이 켜지는 두 가지 방식

1. **자동**: 특정 키워드나 진입점을 인식하면 알아서 작동
   - 예: "워크플로우 보여줘" → `korean-workflow` 자동 활성화
   - 예: `/gv "결제 시스템 만들어줘"` → `project-setup`이 자연스럽게 동반
2. **수동**: Claude Code 표준 plugin 시스템으로 추가 설치 후 호출
   - 예: `/plugin install some-plugin@some-marketplace` → 스킬 등록

## 커맨드 vs 스킬

|           | 커맨드                | 스킬                            |
| --------- | --------------------- | ------------------------------- |
| 실행 방법 | `/이름`으로 직접 입력 | 자동 감지 또는 커맨드가 호출    |
| 역할      | 사용자 입력 진입점    | 실제 작업 로직                  |
| 파일 위치 | `commands/*.md`       | `skills/*/SKILL.md`             |
| 예시      | `/gv`, `/gv:status`   | `multi-review`, `project-setup` |

---

## 역할별 추천 자연어 / 에이전트

v2에서는 별도의 `preset` 명령 없이 역할별로 자연어 진입과 에이전트 활용을 조합합니다.

| 역할      | 추천 자연어 진입                                       | 추천 에이전트/스킬                 |
| --------- | ------------------------------------------------------ | ---------------------------------- |
| 개발자    | `/gv "리팩토링해줘"`, `/gv "이 PR 리뷰"`               | tdd-coach-kr, code-reviewer-kr     |
| PM/기획자 | `/gv "기획서 다듬어줘"`, `/gv "이슈 정리"`             | korean-workflow, doc-reviewer-kr   |
| 디자이너  | `/gv "이 CSS 접근성 점검"`                             | accessibility-checker, mentor-kr   |
| 리서처    | `/gv "X vs Y 비교"`, `/gv "이 데이터 분석"`            | data-analyst-kr, content-editor-kr |
| 학생      | `/gv "이 코드 어떻게 동작해?"`, `/gv "TDD로 만들어줘"` | beginner-guide, mentor-kr          |
