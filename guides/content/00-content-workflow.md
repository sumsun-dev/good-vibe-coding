# 콘텐츠 워크플로우 가이드

## 개요

Technical Writer, Content Editor 역할의 콘텐츠 제작 가이드입니다.

## 콘텐츠 프로세스

### 1단계: 대상 독자 정의

- 기술 수준 (초급/중급/고급)
- 사용 맥락 (온보딩, 레퍼런스, 튜토리얼)
- 언어 및 톤 (한국어, 존댓말/반말)

### 2단계: 구조 설계

```
제목 → 개요 → 전제조건 → 본문 → 예시 → 다음 단계
```

문서 유형별 템플릿:

| 유형     | 구조             | 예시                     |
| -------- | ---------------- | ------------------------ |
| 퀵스타트 | 5분 내 첫 결과물 | 설치 → 실행 → 확인       |
| 튜토리얼 | 단계별 따라하기  | Step 1 → Step 2 → ...    |
| 레퍼런스 | API/커맨드 목록  | 함수명 + 파라미터 + 예시 |
| 가이드   | 개념 + 실전 활용 | 개요 → 원리 → 적용       |

### 3단계: 초안 작성

- 코드 예시는 실행 가능해야 함
- 한국어 문서에서 기술 용어는 영어 유지 (예: API, CLI, JSON)
- 스크린샷보다 텍스트 기반 설명 우선

### 4단계: 리뷰

- 기술 정확성: 엔지니어 리뷰
- 가독성: 대상 독자 수준 확인
- 완성도: 누락된 단계, 깨진 링크 확인

## 스타일 가이드

- 문장은 짧고 명확하게
- 능동태 사용 (피동태 지양)
- 코드 블록에 언어 태그 필수 (`javascript, `bash)
- 파일 경로는 코드 서식(`path/to/file`) 사용

## 도구 활용

- Read: 기존 문서 분석
- Write/Edit: 문서 작성 및 수정
- Grep: 코드에서 예시 추출
