# 코드 리뷰 가이드

## 코드 리뷰란?

코드 리뷰는 작성한 코드를 다른 사람(또는 AI)이 검토하여 품질을 높이는 과정입니다. Good Vibe Coding에서는 `code-reviewer-kr` 에이전트가 체계적인 코드 리뷰를 수행합니다.

---

## code-reviewer-kr 에이전트 활용법

### 기본 사용
```
> 이 코드를 리뷰해줘
> @code-reviewer-kr 이 PR을 검토해줘
> @code-reviewer-kr src/auth/ 폴더의 변경사항을 리뷰해줘
```

### 리뷰 결과 읽기

에이전트는 심각도별로 피드백을 제공합니다:
- **Critical**: 반드시 수정해야 하는 보안/버그 이슈
- **Warning**: 권장하는 개선 사항
- **Suggestion**: 선택적 개선 제안

## 리뷰 체크리스트

### 1. 보안
- [ ] 하드코딩된 API 키, 비밀번호, 토큰 없는가
- [ ] SQL 인젝션 방지 (파라미터화된 쿼리 사용)
- [ ] XSS 방지 (사용자 입력 이스케이프)
- [ ] CSRF 토큰 확인
- [ ] 민감한 데이터가 로그에 노출되지 않는가

### 2. 성능
- [ ] 불필요한 반복문/연산 없는가
- [ ] N+1 쿼리 문제 없는가
- [ ] 메모리 누수 가능성 없는가
- [ ] 적절한 캐싱 적용되었는가
- [ ] 큰 데이터의 페이지네이션 처리

### 3. 확장성
- [ ] 단일 책임 원칙을 따르는가
- [ ] 의존성 주입이 적용되었는가
- [ ] 인터페이스로 계약이 정의되었는가
- [ ] 매직 넘버가 상수로 추출되었는가
- [ ] 설정값이 환경변수로 분리되었는가

### 4. 가독성
- [ ] 네이밍 컨벤션이 일관적인가 (camelCase, PascalCase 등)
- [ ] 함수가 50줄 이내인가
- [ ] 중첩 깊이가 4단계 이내인가
- [ ] 복잡한 로직에 설명이 있는가
- [ ] early return 패턴을 사용하는가

### 5. 테스트
- [ ] 새 기능에 대한 테스트가 있는가
- [ ] 엣지 케이스가 테스트되었는가
- [ ] 테스트가 독립적인가
- [ ] mock이 적절히 사용되었는가
- [ ] 커버리지 80% 이상인가

## 리뷰 요청 팁

### 효과적인 리뷰 요청
```
> @code-reviewer-kr
> 이번에 추가한 인증 미들웨어를 리뷰해줘.
> 주요 변경: src/middleware/auth.js (신규)
> 관심 포인트: JWT 검증 로직의 보안성
```

### 특정 관점으로 리뷰 요청
```
> @code-reviewer-kr 보안 관점에서만 리뷰해줘
> @code-reviewer-kr 성능 이슈가 있는지 확인해줘
> @code-reviewer-kr 이 코드의 테스트 가능성을 평가해줘
```

## 개발자 워크플로우에서의 위치
```
[1.기획] → [2.Side Impact] → [3.TDD] → [4.구현] → [5.검증] → [6.리뷰]
                                                                 ^^^여기
```

코드 리뷰는 워크플로우의 마지막 단계입니다. 검증(build, lint, test)이 모두 통과한 후에 리뷰를 진행합니다.
