# 리서치 워크플로우 가이드

## 개요

리서처 역할(Market, Business, Tech, Design)의 리서치 수행 가이드입니다.

## 리서치 프로세스

### 1단계: 리서치 스코프 정의

- 프로젝트 요구사항에서 리서치 질문 도출
- 1차/2차 자료 소스 식별
- 분석 프레임워크 선택

### 2단계: 데이터 수집

- **Market Researcher**: 시장 규모, 경쟁사, TAM/SAM/SOM
- **Business Researcher**: 비즈니스 모델, 수익 구조, 성장 전략
- **Tech Researcher**: 기술 스택 비교, 벤치마크, 오픈소스 생태계
- **Design Researcher**: 사용자 인터뷰, UX 벤치마크, 접근성 가이드라인

### 3단계: 분석 및 인사이트

```
데이터 수집 → 패턴 식별 → 가설 수립 → 검증 → 인사이트 도출
```

### 4단계: 보고서 작성

리서치 결과를 구조화된 형식으로 정리:

| 섹션   | 내용                             |
| ------ | -------------------------------- |
| 요약   | 핵심 발견 3-5개                  |
| 방법론 | 분석 프레임워크, 데이터 소스     |
| 분석   | 상세 분석 결과                   |
| 제안   | 실행 가능한 제안 사항            |
| 제약   | 분석의 한계, 추가 조사 필요 영역 |

## 도구 활용

- WebSearch: 최신 시장 데이터, 트렌드 조사
- WebFetch: 특정 URL의 상세 정보 수집
- Read/Glob: 프로젝트 내부 문서 분석

## 팀 협업

- **Market/Business Researcher**: Tier 1에서 전략적 리서치 수행 (CTO, PO와 동시, priority 2)
- **Tech/Design Researcher**: Tier 3에서 기술/디자인 검증 수행 (QA, Security와 동시, priority 5)
- 기본 설정(`parallelTiers=true`)에서는 모든 에이전트가 동시 실행되므로 Tier 구분 없이 병렬 분석
- 분석 결과는 종합 단계에서 기획서에 반영
- 크로스 리뷰 시 도메인 전문성 기반 피드백 제공
