# PRD 작성 가이드

## PRD란?

PRD(Product Requirements Document)는 제품의 요구사항을 정리한 문서입니다. Claude Code를 활용하면 PRD 작성을 체계적이고 빠르게 할 수 있습니다.

---

## Claude Code로 PRD 작성하기

### 시작하기

```
> 새로운 PRD를 작성해줘: [기능명]
> 사용자 인증 기능에 대한 PRD를 만들어줘
```

### doc-reviewer-kr 에이전트 활용

```
> @doc-reviewer-kr 이 PRD를 검토해줘
```

## PRD 템플릿 구조

### 1. 개요

```markdown
# [기능명] PRD

## 개요

- **작성일**: YYYY-MM-DD
- **작성자**: [이름]
- **상태**: 초안 / 검토 중 / 승인됨
- **우선순위**: P0 (긴급) / P1 (높음) / P2 (보통) / P3 (낮음)

## 배경

왜 이 기능이 필요한가? 어떤 문제를 해결하는가?

## 목표

이 기능으로 달성하려는 구체적인 목표 (측정 가능하게)
```

### 2. 요구사항

```markdown
## 기능 요구사항

### 필수 (Must Have)

- [ ] 요구사항 1
- [ ] 요구사항 2

### 권장 (Should Have)

- [ ] 요구사항 3

### 선택 (Nice to Have)

- [ ] 요구사항 4

## 비기능 요구사항

- 성능: 응답 시간 200ms 이내
- 보안: 인증 토큰 암호화
- 확장성: 동시 사용자 1000명 지원
```

### 3. 사용자 스토리

```markdown
## 사용자 스토리

### 스토리 1: 로그인

- **사용자**: 일반 사용자
- **목표**: 이메일과 비밀번호로 로그인
- **상세**:
  1. 로그인 페이지에 접속한다
  2. 이메일과 비밀번호를 입력한다
  3. "로그인" 버튼을 클릭한다
  4. 대시보드로 이동한다
- **예외**: 비밀번호 5회 오류 시 계정 잠금
```

### 4. 범위와 일정

```markdown
## 범위

### 포함

- 이메일/비밀번호 로그인
- 비밀번호 재설정

### 제외 (이번 버전)

- 소셜 로그인
- 2FA

## 일정

| 마일스톤    | 기한 | 담당     |
| ----------- | ---- | -------- |
| PRD 승인    | W1   | PM       |
| 디자인 완료 | W2   | 디자이너 |
| 개발 완료   | W4   | 개발팀   |
| QA          | W5   | QA팀     |
```

## PRD 작성 팁

### Claude Code 활용 예시

```
> 이 PRD에서 빠진 내용이 있는지 확인해줘
> 사용자 스토리를 3개 더 추가해줘
> 비기능 요구사항을 구체적인 수치로 바꿔줘
> 이 PRD를 이슈 티켓으로 분해해줘
```

### 좋은 PRD의 특징

- **구체적**: "빠르게" 대신 "200ms 이내"
- **측정 가능**: 성공 기준이 숫자로 표현됨
- **완전**: 정상 케이스와 예외 케이스 모두 포함
- **간결**: 불필요한 설명 없이 핵심만
