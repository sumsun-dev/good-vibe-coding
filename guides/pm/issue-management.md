# 이슈 관리 가이드

## 이슈 관리란?

이슈 관리는 프로젝트의 작업 단위(버그, 기능, 개선)를 추적하고 관리하는 과정입니다. Claude Code를 활용하면 이슈 생성, 분류, 추적을 자동화할 수 있습니다.

---

## Claude Code로 이슈 관리하기

### 이슈 생성
```
> PRD를 기반으로 GitHub 이슈를 만들어줘
> 이 버그를 이슈로 등록해줘
> 이번 스프린트의 이슈 목록을 만들어줘
```

### GitHub Issues 연동
Claude Code는 `gh` CLI를 통해 GitHub Issues와 직접 연동됩니다:

```
> gh issue list로 현재 이슈를 보여줘
> 새 이슈를 만들어줘: "로그인 페이지 반응형 수정"
> #42 이슈에 댓글을 달아줘
```

---

## 이슈 작성 규칙

### 제목 규칙
- 행동 동사로 시작: "추가", "수정", "개선", "제거"
- 구체적이고 검색 가능하게 작성

```
로그인 문제
[버그] 소셜 로그인 시 리다이렉트 URL 누락으로 로그인 실패
```

### 이슈 템플릿

**기능 요청:**
```markdown
## 기능 설명
[기능에 대한 간단한 설명]

## 사용자 스토리
[사용자]로서, [목표]를 하고 싶다. 왜냐하면 [이유]이기 때문이다.

## 상세 요구사항
- [ ] 요구사항 1
- [ ] 요구사항 2

## 디자인/참고 자료
[링크 또는 스크린샷]

## 우선순위
P0 / P1 / P2 / P3
```

**버그 리포트:**
```markdown
## 증상
[무엇이 잘못되었는지]

## 재현 단계
1. [단계 1]
2. [단계 2]
3. [단계 3]

## 기대 동작
[어떻게 동작해야 하는지]

## 실제 동작
[실제로 어떻게 동작하는지]

## 환경
- OS:
- 브라우저:
- 버전:
```

---

## 우선순위 체계

| 등급 | 의미 | 예시 | 대응 기한 |
|------|------|------|----------|
| P0 | 긴급 (서비스 중단) | 결제 불가, 데이터 손실 | 즉시 |
| P1 | 높음 (핵심 기능 장애) | 로그인 실패, 주요 기능 오류 | 24시간 |
| P2 | 보통 (불편하지만 우회 가능) | UI 깨짐, 느린 응답 | 이번 스프린트 |
| P3 | 낮음 (개선 사항) | 오타, 스타일 개선 | 백로그 |

## 이슈 관리 자동화

### Claude Code 활용 예시

**PRD → 이슈 분해:**
```
> 이 PRD를 실행 가능한 이슈 단위로 분해해줘
> 각 이슈에 예상 소요 시간과 우선순위를 매겨줘
```

**이슈 상태 업데이트:**
```
> #42 이슈를 완료 처리해줘
> 이번 주에 완료된 이슈를 정리해줘
> 블로커가 있는 이슈를 찾아줘
```

**스프린트 리포트:**
```
> 이번 스프린트의 진행 상황을 요약해줘
> 완료/진행중/대기 이슈 수를 정리해줘
```
