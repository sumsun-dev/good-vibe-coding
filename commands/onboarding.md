# /onboarding - Claude Code 초기 설정 마법사

한국어 인터랙티브 온보딩을 시작합니다. 아래 단계를 순서대로 진행하세요.

## 실행 방법
사용자가 `/onboarding`을 입력하면 이 커맨드가 실행됩니다.

## 진행 절차

### Step 1: 환영 + 환경 감지
다음을 확인하고 사용자에게 알려주세요:
- Claude Code 버전 (`claude --version`)
- 운영체제
- 기존 `~/.claude/CLAUDE.md` 존재 여부
- 기존 설정이 있으면 백업 여부를 확인

```
안녕하세요! Claude Code 설정을 도와드릴게요.

현재 환경을 확인합니다...
✅ Claude Code v{버전}
✅ {운영체제}
{기존 설정 상태}

몇 가지 질문에 답해주시면 최적의 설정을 만들어 드릴게요.
```

### Step 2: 역할 선택
AskUserQuestion 도구를 사용하여 역할을 묻습니다:

질문: "어떤 일을 주로 하시나요?"
선택지:
1. **개발** - 풀스택 / 프론트엔드 / 백엔드 / 데이터
2. **기획/관리** - PM / PO / 프로젝트 매니저
3. **디자인** - UI/UX 디자이너 / 웹 퍼블리셔
4. **리서치/분석** - UX 리서처 / 데이터 분석가
5. **콘텐츠/마케팅** - 블로거 / 마케터 / 카피라이터
6. **학습** - 학생 / 입문자 / 직무전환자

역할 → 프리셋 매핑:
- 개발 → `developer`
- 기획/관리 → `pm`
- 디자인 → `designer`
- 리서치/분석 → `researcher`
- 콘텐츠/마케팅 → `content-creator`
- 학습 → `student`

### Step 3: 업무/도구 세부 선택
역할에 따라 추가 질문을 합니다:

**개발자 선택 시:**
- 질문: "주로 사용하는 기술 스택은?"
- 선택지: Next.js + Supabase / React + Node.js / Python + FastAPI / 기타

**PM/기획자 선택 시:**
- 질문: "주로 하는 업무는? (복수 선택)"
- 선택지: PRD/기획서 작성 / 이슈/티켓 관리 / 데이터 분석/리포트 / 회의록/문서 정리

**디자이너 선택 시:**
- 질문: "주요 작업 영역은?"
- 선택지: UI/UX 디자인 / 웹 퍼블리싱 / 디자인 시스템 관리 / 접근성 검수

**리서처 선택 시:**
- 질문: "주요 리서치 영역은?"
- 선택지: UX 리서치 / 데이터 분석 / 시장 조사 / 학술 연구

**콘텐츠/마케팅 선택 시:**
- 질문: "주로 작성하는 콘텐츠는?"
- 선택지: 블로그/기술 문서 / SNS 콘텐츠 / 뉴스레터 / 광고 카피

**학습 선택 시:**
- 질문: "현재 학습 단계는?"
- 선택지: 프로그래밍 입문 / 특정 기술 학습 중 / 프로젝트 실습 / 직무 전환 준비

### Step 4: 워크플로우 선택
역할에 맞는 워크플로우 옵션을 제시합니다.

(워크플로우 선택지는 아래 참조)


**개발자:**
- 풀 워크플로우 (기획→TDD→검증→리뷰) (Recommended)
- TDD 중심
- 간소화

**PM/기획자:**
- 체계적 (템플릿 기반) (Recommended)
- 유연 (자유 형식)
- 애자일 (스프린트 중심)

**디자이너:**
- 컴포넌트 기반 (디자인 시스템 활용) (Recommended)
- 자유 디자인
- 접근성 중심

**리서처:**
- 체계적 리서치 (가설→수집→분석) (Recommended)
- 탐색적 분석
- 리포트 중심

**콘텐츠/마케팅:**
- 기획→작성→편집→발행 (Recommended)
- 빠른 작성
- SEO 중심

**학습:**
- 단계별 학습 (개념→실습→복습) (Recommended)
- 프로젝트 기반
- 자유 탐색

### Step 4A: 팀원 스타일 선택

각 에이전트에 대해 AskUserQuestion으로 성격 변형을 선택합니다.
첫 번째 옵션에는 "(Recommended)" 표시.
"기본 스타일로 진행" 옵션도 제공하여 스킵 가능.

**진행 방식:**
1. 역할 프리셋에 포함된 에이전트 목록을 가져옵니다
2. `personality-builder`의 `getPersonalityVariants()`로 각 에이전트의 변형을 조회합니다
3. AskUserQuestion으로 각 에이전트의 스타일을 묻습니다:

예시 (개발자 역할):
```
질문: "코드 리뷰어의 스타일을 선택해 주세요"
header: "리뷰 스타일"
options:
  - label: "꼼꼼한 검토자 🔍 (Recommended)"
    description: "세심하게 체크하고 개선점을 정확히 제시합니다"
  - label: "친절한 멘토 🤝"
    description: "좋은 점을 먼저 짚고, 개선점을 부드럽게 제안합니다"
  - label: "기본 스타일로 진행"
    description: "기본값을 사용합니다"
```

4. 선택 결과를 `choices.personalities` 객체에 저장합니다
5. 모든 에이전트 선택 완료 후 팀 소개 메시지를 보여줍니다:

```
🎉 당신의 팀이 구성되었습니다!

  준영 (꼼꼼한 검토자) 🔍 — 코드 리뷰어
  하윤 (엄격한 코치) 🎯 — TDD 코치
```

### Step 5: 설정 생성 확인
생성할 파일 목록을 보여주고 확인을 받습니다.

확인 후 `config-generator` 스킬을 사용하여 설정 파일을 생성합니다.

완료 메시지:
```
✅ 모든 설정이 완료되었습니다!

💡 다음 단계:
  /learn     → 내 역할에 맞는 Claude Code 활용법 배우기
  /my-config → 현재 설정 확인하기
  /add-skill → 추가 스킬 설치하기
  /add-agent → 추가 에이전트 설치하기
```
