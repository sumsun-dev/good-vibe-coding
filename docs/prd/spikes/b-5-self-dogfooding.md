## 검증 대상

`/gv <자연어>` → dispatch → task-router → task-graph-presets → task-graph-runner → 실제 LLM action까지의 end-to-end 흐름이 v2 자체 마이그레이션 작업에 사용 가능한가?

## 결과 요약

**부분 가능**. 그래프 골격, 분류, 실행 드라이버, 라이브 패널, opt-in 예산까지 모두 동작하지만, 실제 도그푸딩으로 self-마이그레이션 작업을 돌리려면 다음 통합이 더 필요:

| 영역                                      | 상태                                                             |
| ----------------------------------------- | ---------------------------------------------------------------- |
| 자연어 진입 (`/gv`)                       | ✅ 동작 (#248)                                                   |
| 5개 작업 유형 분류                        | ✅ 100% 정확도 (#240)                                            |
| 5개 그래프 동적 실행                      | ✅ runGraph + 5개 builder (#252-#260)                            |
| 라이브 패널 출력                          | ✅ stdout markdown (#246)                                        |
| opt-in 예산 임계                          | ✅ /gv:cost (#250)                                               |
| 실제 LLM 호출                             | ⚠️ callLLMWithFallback 인프라 OK, 실제 사용은 사용자 API 키 필요 |
| 코드 파일 쓰기 (`code-materializer` 통합) | ❌ 미통합 — code 그래프의 `materializing`이 LLM 요약만           |
| 빌드/테스트 검증 (`execution-verifier`)   | ❌ 미통합                                                        |
| 다중 라운드 토론 (plan)                   | ❌ 단일 LLM 호출 (NEXT_ROUND graph guard만 사용)                 |
| CEO 입력 통합 (escalating)                | ❌ LLM 자동 결정 (실제 CEO UI 없음)                              |

## 도그푸딩 대신 사용한 검증 방법

- **단위 테스트 87개+** (5개 graph builder × 평균 17개 + runner/dispatch/cost/render)
- **mock callLLM**으로 5개 그래프 happy/sad path 모두 시뮬레이션
- **회귀 통과** (전체 2910 passed) — 기존 v1 흐름과 v2 신규 모듈 충돌 없음

mock 기반이라 실제 LLM 응답 품질은 검증 못 했지만, **그래프 진행 정확성과 모듈 간 통합은 충분히 검증**됨.

## Phase C 진입 결정

§13-5 ("자체 도그푸딩 시점 보류")의 결정 시점이다. 두 옵션:

| 옵션                     | 설명                                                                                      |
| ------------------------ | ----------------------------------------------------------------------------------------- |
| **A. Phase C 즉시 진입** | mock 기반 검증으로 충분. 실제 LLM 통합/도그푸딩은 v2 릴리즈 후 사용자 환경에서            |
| **B. 도그푸딩 먼저**     | code-materializer/execution-verifier 통합까지 마치고 self-마이그레이션 시도. Phase C 지연 |

**권고**: **A 옵션**. 현재 mock 검증 + 87개 단위 테스트로 그래프 정확성은 충분히 보장됨. 실제 LLM 응답 품질은 사용자 API 키 + 실 사용 데이터로만 평가 가능하므로, Phase C에서 v1 슬래시 제거하고 v2 릴리즈 → 실 사용 누적이 자연스러운 도그푸딩 경로.

`code-materializer/execution-verifier` 통합은 v2 릴리즈 후 후속 마이너 버전(v2.1+)에서 점진 추가.
