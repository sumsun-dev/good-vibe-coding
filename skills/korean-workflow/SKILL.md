# Korean Workflow

한국어 워크플로우 가이드를 제공합니다.

## 트리거
- 사용자가 워크플로우 관련 질문 시 자동 활성화
- "다음 단계", "지금 뭐 해야 해?", "워크플로우" 키워드 감지

## 입력
- 현재 역할 (`~/.claude/CLAUDE.md`에서 확인)
- 현재 진행 중인 작업 (선택)

## 플로우

### 1단계: 역할 확인
`~/.claude/CLAUDE.md` 파일을 읽어 현재 역할을 파악합니다.

### 2단계: 워크플로우 표시
역할별 워크플로우 단계를 시각적으로 보여줍니다.

### 6개 역할별 워크플로우

**개발자:**
```
[1.기획] → [2.Side Impact] → [3.TDD] → [4.구현] → [5.검증] → [6.리뷰]
   📋          🔍              🔴🟢        💻          ✅          👀
```
- 1단계: Plan Mode로 구현 계획 → 사용자 승인
- 2단계: 영향 범위 분석 (변경 대상 파일, 의존성 파악)
- 3단계: 테스트 먼저 작성 (RED → GREEN → REFACTOR)
- 4단계: 코드 구현 + 리팩토링
- 5단계: build, lint, typecheck, test 전체 통과
- 6단계: code-reviewer 에이전트로 코드 리뷰

**PM / 기획자:**
```
[1.분석] → [2.기획서] → [3.검토] → [4.이슈관리] → [5.추적]
   🔍         📝           👀          🎫            📊
```
- 1단계: 요구사항 분석 + 사용자 스토리 정의
- 2단계: PRD/기능 명세서 초안 작성
- 3단계: 문서 리뷰 + 피드백 반영
- 4단계: 티켓 생성 + 우선순위 지정
- 5단계: 진행 상황 모니터링 + 리포트

**디자이너:**
```
[1.분석] → [2.구조설계] → [3.구현] → [4.검증] → [5.QA]
   🔍         🏗️            🎨        ♿          ✅
```
- 1단계: 요구사항 확인 + 레퍼런스 조사
- 2단계: 컴포넌트 구조 정의
- 3단계: HTML/CSS 마크업
- 4단계: 접근성 + 반응형 체크
- 5단계: 디자인 QA

**리서처 / 분석가:**
```
[1.정의] → [2.수집] → [3.분석] → [4.해석] → [5.보고]
   🎯        📦         🔬         💡         📄
```
- 1단계: 리서치 질문 설정
- 2단계: 소스 확보 + 데이터 정리
- 3단계: 데이터 처리 + 패턴 발견
- 4단계: 인사이트 도출
- 5단계: 리포트 작성 + 공유

**콘텐츠 크리에이터:**
```
[1.기획] → [2.구조화] → [3.작성] → [4.편집] → [5.발행]
   🎯        📋           ✏️         🔧         🚀
```
- 1단계: 주제 선정 + 키워드 리서치
- 2단계: 아웃라인 작성
- 3단계: 초안 작성
- 4단계: 교정 + SEO 최적화
- 5단계: 최종 검토 + 게시

**학생 / 입문자:**
```
[1.학습] → [2.관찰] → [3.실습] → [4.응용] → [5.복습]
   📚        👀         ⌨️         🔧         📝
```
- 1단계: 개념 이해
- 2단계: 예시 코드 분석
- 3단계: 직접 따라하기
- 4단계: 변형해서 만들기
- 5단계: 배운 내용 정리

### 3단계: 현재 위치 표시
사용자의 현재 작업을 기반으로 워크플로우에서 현재 위치를 강조합니다.

```
[1.기획] → [2.Side Impact] → [3.TDD] → [4.구현] → [5.검증] → [6.리뷰]
                                ^^^현재
```

### 4단계: 다음 행동 추천
현재 단계에서 사용할 수 있는 커맨드, 스킬, 에이전트를 추천합니다.

## 중요 규칙
- 항상 한국어로 안내
- 워크플로우 단계를 건너뛰지 않도록 안내 (순서 중요)
- 각 단계 완료 기준을 명확히 제시
- 사용자의 역할에 맞지 않는 워크플로우는 제안하지 않기
